# 交易连接器模块

## 概述

交易连接器模块封装了 10+ 券商/交易所的 SDK，提供统一的交易接口，包括账户查询、订单管理、持仓管理、行情获取等功能。

**源文件**: `agent/src/trading/connectors/` 目录

## 核心架构

### 1. 连接器统一接口

每个券商连接器包含三个核心文件：

| 文件 | 职责 |
|------|------|
| `sdk.py` | SDK 封装，实现 `BaseSDK` 接口 |
| `classification.py` | 标的分类（股票/期货/期权/加密货币等） |
| `profiles.py` | 券商配置 Profile 加载 |

统一服务入口在 `agent/src/trading/service.py`，通过 `connector_profile_id_for_broker()` 查找对应连接器。

|------|------|----------|

| Dhan | `agent/src/trading/connectors/dhan/` | 印度股票 |

### 3. 类型系统

`agent/src/trading/types.py` 定义了统一的交易类型：

- `OrderSide`: 买卖方向
- `OrderType`: 订单类型（市价/限价/止损）
- `TimeInForce`: 有效期（当日/永久成交前有效）
- `Position`: 持仓数据结构
- `TradeRecord`: 成交记录

### 4. 连接器 Profile

`agent/src/trading/profiles.py` 管理券商配置文件，每个券商在 `~/.vibe-trading/live/<broker>/profiles/` 目录下有独立配置文件，包含 API Key、Secret、Endpoint 等敏感信息。

**使用场景**:
```python
from src.trading.service import connector_profile_id_for_broker
from src.trading.connectors.alpaca.sdk import AlpacaSDK

profile_id = connector_profile_id_for_broker("alpaca")
sdk = AlpacaSDK(profile_id=profile_id)
orders = sdk.get_orders(status="open")
```
