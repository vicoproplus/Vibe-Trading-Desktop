# 实盘交易模块

## 概述

实盘交易模块是 Vibe Trading 的自动化交易执行系统，涵盖安全订单门控、交易授权（Mandate）、紧急暂停（Halt）、审计日志、运行时调度等核心功能。

**源文件**: `agent/src/live/` 目录

## 核心组件

### 1. 订单门控 — SDK Order Gate

`SDKOrderGate` 定义在 `agent/src/live/sdk_order_gate.py`，是实盘交易的安全闸门：

- 检查 Mandate 是否授权当前交易
- 检查 Halt 状态（全局 kill-switch）
- 验证交易额度/仓位限制
- 拒绝未授权交易（fail-closed 设计）
- 记录每个订单的审计轨迹

**使用场景**:
```python
from src.live.sdk_order_gate import SDKOrderGate
gate = SDKOrderGate()
result = gate.check_order({
    "broker": "alpaca",
    "symbol": "AAPL",
    "side": "buy",
    "qty": 100,
    "type": "market"
})
```

### 2. 交易授权 — Mandate

定义在 `agent/src/live/mandate/` 目录：

- **`model.py`**：Mandate 数据模型（交易标的、额度、风控参数）
- **`store.py`**：Mandate 持久化存储
- **`commit.py`**：Mandate 提交与确认

Mandate 包含：
- 允许交易的标的列表
- 单笔/日/总交易额度上限
- 风险等级（RiskTier）
- 杠杆限制
- 衍生品类型限制

**使用场景**:
```bash
# CLI 提交 Mandate
vibe-trading live propose-mandate --limits-file mandate.yaml
vibe-trading live commit-mandate --proposal-id <id>
```

### 3. 紧急暂停 — Halt

`halt.py`（`agent/src/live/halt.py`）实现全局 kill-switch：

- `trip_halt(reason)`：触发热停，所有交易立即停止
- `clear_halt()`：恢复交易
- `halt_flag_set()`：查询暂停状态

Halt 状态通过 `LIVE_HALT` 文件持久化（`agent/src/live/paths.py`），即使重启也保持状态。

**使用场景**:
```bash
vibe-trading live halt --reason "市场异常波动"
vibe-trading live resume
```

### 4. 审计日志

`audit.py`（`agent/src/live/audit.py`）记录每次交易操作：

- `write_live_action()`：写入审计记录
- 记录内容：时间、券商、操作类型、标的、数量、价格、状态
- 审计日志存储在 `~/.vibe-trading/live/<broker>/audit/` 目录

### 5. 运行时系统

定义在 `agent/src/live/runtime/` 目录：

| 组件 | 文件 | 职责 |
|------|------|------|
| `Scheduler` | `agent/src/live/runtime/scheduler.py` | 定时任务调度器 |
| `Runner` | `agent/src/live/runtime/runner.py` | 实盘运行器（Agent 持续运行模式） |
| `Liveness` | `agent/src/live/runtime/liveness.py` | 活跃性探针 |
| `Reconcile` | `agent/src/live/runtime/reconcile.py` | 订单/持仓对账 |
| `JobStore` | `agent/src/live/runtime/jobstore.py` | 任务持久化 |
| `Flatten` | `agent/src/live/runtime/flatten.py` | 仓位平仓/减仓 |
| `Triggers` | `agent/src/live/runtime/triggers.py` | 触发条件定义 |

### 6. 执行强制

`enforcement.py`（`agent/src/live/enforcement.py`）提供 L6 层级的强制规则执行：

- 检查交易额度
- 验证持仓限制
- 确保交易只在允许时间段进行

## 安全体系

实盘模块的安全设计遵循 fail-closed 原则：

1. **默认拒绝**：所有交易必须通过 Mandate 明确授权
2. **多层验证**：Gate → Enforcement → Broker Connector
3. **全局熔断**：Halt 机制覆盖所有券商
4. **完整审计**：每笔操作可追溯
